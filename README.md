# ExamOCR AES

ExamOCR AES is an English essay OCR and scoring pipeline. It supports three scoring modes:

- `baseline`: grade directly with the official rubric.
- `autoscore`: extract rubric evidence first, then score with tier constraints.
- `ASRO`: apply optimized adaptation rules generated by ASRO training.

The pipeline is dataset-driven. Official question metadata and rubrics are loaded from `json_path -> subjective_question[qid].rubric`; prompts should not contain task-specific rubrics.

## Setup

Install dependencies:

```bash
pip install -r requirements.txt
```

Set an API key through environment variables, a local `.env` file, or YAML config:

```dotenv
OPENROUTER_API_KEY=your_api_key
```

`.env` is ignored by git and loaded by the Python entrypoints. The API wrapper is OpenAI-compatible and uses `base_url` from the config. Do not commit real API keys in YAML files.

## Dataset Format

The real dataset is a JSON object keyed by paper ID. Each paper contains:

- `subject`
- `blank_sheet`
- `subjective_question`
- `student_answer`

Important fields:

- Student answers are image paths, not OCR text.
- Human labels are in `student_answer[*].grading[qid]`.
- Code should use `reference_analysis` with `.get(...)` because it may be absent.

For schema inspection, use `data/processed_5.0pct_66_67/processed_5.0pct_66_67/dataset_sample_schema.json`. Do not inspect the full `dataset.json` during development unless explicitly required.

## Main Pipeline

Run OCR and scoring with:

```bash
python main.py --config configs/demo_baseline.yaml
```

Override the config section if needed:

```bash
python main.py --config configs/full_pipeline.yaml --section pipeline
```

Limit the number of processed samples:

```bash
python main.py --config configs/full_pipeline.yaml --num 5
```

### Pipeline Config

`main.py` reads the `pipeline` section. Common keys:

| Key | Purpose |
| --- | --- |
| `mode` | `baseline`, `autoscore`, or `ASRO` |
| `input_dir` | directory containing answer images or OCR `.txt` files |
| `out_dir` | output directory for CSV results and error reports |
| `json_path` | dataset metadata and human scores |
| `dataset_name` | top-level paper ID in the dataset |
| `target_id` | question filter, for example `"66"` or `"66,67"` |
| `rules_file` | optimized ASRO rules file used in `ASRO` mode |
| `num` | optional sample limit |
| `workers` | parallel worker count |
| `enable_ocr` | run OCR before scoring |
| `force_ocr` | regenerate OCR text even when `.txt` exists |
| `ocr_only` | run OCR and skip scoring |
| `debug` | write OCR comparison images |
| `max_score` | total score for the target question |
| `tier_count` | number of coarse scoring tiers |
| `strict` | fail fast instead of collecting failed samples |

Outputs are written under `out_dir`:

- `results_{dataset_name}.csv`
- `error_report_{dataset_name}.csv` when any samples fail

The console prints QWK and MAE when human labels are available.

## OCR Only

Standalone OCR:

```bash
python scripts/run_ocr.py --input_dir data/sample/images --config configs/demo_baseline.yaml --limit 20
```

`--num` is accepted as an alias for `--limit`. OCR writes `.txt` files next to the source images.

Pipeline OCR-only mode:

```yaml
pipeline:
  enable_ocr: true
  ocr_only: true
```

## ASRO Training

Run ASRO guideline optimization:

```bash
python ASRO/train.py --config configs/demo_asro_train.yaml
```

`ASRO/train.py` reads the `asro_train` section. Common keys:

| Key | Purpose |
| --- | --- |
| `data_dir` | OCR text directory used for training and validation |
| `json_path` | dataset metadata and human scores |
| `dataset_name` | top-level paper ID |
| `target_q` | target question ID |
| `train_n`, `val_n` | optional split sizes |
| `val_ratio` | validation ratio when split sizes are not fixed |
| `T` | ASRO rounds |
| `B` | minibatch size |
| `K` | number of error modes refined per round |
| `max_workers` | parallel scoring workers |
| `grader_model` | grader model |
| `reflector_model` | reflector/refiner model |
| `output_dir` | optimized guideline output directory |
| `ocr_before_train` | run OCR before training if text files are missing |

The initial ASRO guideline is built from:

- `subjective_question[qid].question`
- `subjective_question[qid].rubric`
- `subjective_question[qid].reference_answer`
- `subjective_question[qid].reference_analysis`

Training writes optimized guideline JSON and metrics into `output_dir`.

## Demo Commands

```bash
bash scripts/run_demo_baseline.sh
bash scripts/run_demo_autoscore.sh
bash scripts/run_demo_asro_train.sh
```

You can pass a config path or a numeric sample limit:

```bash
bash scripts/run_demo_baseline.sh 5
bash scripts/run_demo_autoscore.sh configs/demo_autoscore.yaml 10
```

`PYTHON_CMD` can be used to choose a Python environment:

```bash
PYTHON_CMD="conda run --no-capture-output -n env python" bash scripts/run_demo_baseline.sh
```

## Smoke Test

The smoke test uses a fake scoring client and does not call external LLM APIs:

```bash
python tests/smoke_test.py --samples-per-paper 6 --target-id "66,67"
```

It creates temporary inputs and outputs under `.smoke_test/`.

## Full Runner

The root runner supports:

```bash
bash run.sh pipeline
bash run.sh train
bash run.sh full
```

Default configs:

- `configs/full_pipeline.yaml`
- `configs/full_asro_train.yaml`

Override the config path:

```bash
bash run.sh pipeline configs/demo_baseline.yaml
```

## Development Notes

- Do not read or grep the real `dataset.json` during development.
- Rubrics must come from `subjective_question[qid].rubric`.
- Generated outputs belong in `results/`, `.smoke_test/`, or configured ASRO output directories.
- Keep local secrets in `.env`, environment variables, or ignored local YAML files.
